# DES12 시스템시험 시나리오 예시

문서명: 시스템시험 시나리오  
대상 기능:

- 당뇨 추적검사 누락 알림
- 패혈증 위험 알림

## 1. 시스템시험 시나리오

| 시나리오 ID | 시나리오명 | 사용자 | 기대 결과 |
|---|---|---|---|
| ST-01 | 외래 진료 중 HbA1c 누락 확인 | 주치의 | 배너 표시, 상세보기 가능 |
| ST-02 | 안저검사 누락 예약 연계 | 코디네이터 | 예약 또는 외부검사 등록 가능 |
| ST-03 | 응급실 패혈증 고위험 경고 | 응급실 의사 | 5초 이내 팝업 표시 |
| ST-04 | 패혈증 경고 후 오더 반영 | 응급실 의사 | 로그에 후속조치 저장 |
| ST-05 | 운영자 KPI 조회 | 운영자 | 발생 건수/수용률/무시율 조회 |

## 2. 대표 시나리오 예시

### ST-01 외래 진료 시나리오

1. 주치의가 외래 환자 차트를 연다.
2. 시스템이 당뇨 관리대상 여부를 판정한다.
3. HbA1c 누락 조건을 만족하면 배너를 표시한다.
4. 주치의가 상세보기 후 검사 오더를 입력한다.
5. 로그에 `ORDERED_TEST`가 기록된다.

### ST-03 응급실 시나리오

1. 환자가 응급실에 등록된다.
2. 간호사가 활력징후를 입력한다.
3. 시스템이 qSOFA를 계산한다.
4. 고위험이면 팝업을 표시한다.
5. 의사가 상세정보를 보고 추가 오더를 수행한다.

## 3. 한 줄 정리

시스템시험의 핵심은  
`사용자 입장에서 CDSS가 실제 업무 흐름 안에서 자연스럽고 안전하게 동작하는지 확인하는 것`이다.
