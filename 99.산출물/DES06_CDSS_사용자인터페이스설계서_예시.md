# DES06 사용자 인터페이스 설계서 예시

문서명: 사용자 인터페이스 설계서  
대상 시스템: 병원행정시스템(HIS) 연계 CDSS  
대상 기능:

- 당뇨 환자 추적검사 누락 알림
- 패혈증 위험 알림

## 1. 문서 목적

본 문서는 CDSS 기능이 사용자 화면에서 어떤 형태로 표시되고, 사용자가 어떤 방식으로 반응하는지를 정의하기 위한 것이다.  
의료 시스템 UI는 단순히 보기 좋게 만드는 것이 아니라, 적절한 시점에 필요한 정보만 제공하여 업무 흐름을 방해하지 않도록 설계되어야 한다.

## 2. UI 설계 원칙

| 원칙 | 설명 |
|---|---|
| Workflow Fit | 진료 흐름을 끊지 않도록 필요한 위치와 강도로만 알림 제공 |
| Explainability | 왜 알림이 떴는지 수치와 기준을 함께 표시 |
| Priority | 당뇨는 배너 중심, 패혈증 고위험은 팝업/강조 중심 |
| Actionability | 사용자가 바로 검사 오더, 상세 확인, 무시사유 입력 가능 |
| Auditability | 확인/무시/조치 결과가 로그로 남아야 함 |

## 3. 화면 목록

| 화면 ID | 화면명 | 사용자 | 주요 기능 |
|---|---|---|---|
| UI-01 | 외래 진료화면 당뇨 알림 배너 | 주치의 | HbA1c 누락 여부 안내 |
| UI-02 | 당뇨 알림 상세 팝업 | 주치의 | 마지막 검사일, 기준 사유, 후속 액션 |
| UI-03 | 예약지원 화면 안저검사 연계 | 코디네이터 | 안저검사 누락 확인 및 예약 |
| UI-04 | 응급실 패혈증 고위험 팝업 | 응급실 의사 | qSOFA 기반 고위험 경고 |
| UI-05 | 패혈증 상세 팝업 | 응급실 의사/병동 의사 | 발생 근거, 권고 조치 |
| UI-06 | 운영자 로그 조회 화면 | 운영자, 감염관리실 | 알림 현황 및 조치율 조회 |

## 4. 화면 상세 설계

---

## UI-01 외래 진료화면 당뇨 알림 배너

### 1) 목적

- 외래 진료 중 당뇨 환자의 추적검사 누락 여부를 자연스럽게 알려준다.

### 2) 표시 조건

- 환자가 당뇨 관리대상으로 식별됨
- 최근 90일 이내 HbA1c 결과가 없음

### 3) 배치 위치

- 진료화면 상단 환자 기본정보 영역 바로 아래

### 4) 표시 요소

| 요소 | 내용 |
|---|---|
| 배너 색상 | 주의 수준의 노란색 또는 주황색 |
| 제목 | `최근 90일 내 HbA1c 결과가 없습니다.` |
| 보조 문구 | `마지막 검사일: 2025-12-10` |
| 버튼 1 | `상세보기` |
| 버튼 2 | `검사 오더` |
| 버튼 3 | `무시` |

### 5) 동작 규칙

- `상세보기` 클릭 시 UI-02 팝업 호출
- `검사 오더` 클릭 시 HbA1c 관련 오더화면으로 이동
- `무시` 클릭 시 사유 선택 팝업 표시

---

## UI-02 당뇨 알림 상세 팝업

### 1) 목적

- 사용자가 알림 발생 근거를 이해하고 적절한 판단을 할 수 있게 한다.

### 2) 표시 정보

| 항목 | 예시 |
|---|---|
| 환자명 | 홍길동 |
| 기준 | 최근 90일 이내 HbA1c 결과 없음 |
| 마지막 검사일 | 2025-12-10 |
| 관리대상 식별 근거 | E11.9 진단, 메트포르민 처방 이력 |
| 권고 액션 | HbA1c 검사 처방 또는 추적계획 수립 |

### 3) 버튼

- `검사 오더`
- `다음에 보기`
- `무시 사유 입력`

---

## UI-03 예약지원 화면 안저검사 연계

### 1) 목적

- 코디네이터가 안저검사 누락 환자를 확인하고 예약 지원을 할 수 있게 한다.

### 2) 표시 조건

- 당뇨 관리대상
- 최근 1년 안저검사 기록 없음

### 3) 표시 항목

| 항목 | 예시 |
|---|---|
| 환자명 | 홍길동 |
| 최근 안저검사일 | 없음 |
| 상태 | 예약 권고 |
| 예약 버튼 | 안과 예약 |
| 결과 등록 버튼 | 외부검사 등록 |

---

## UI-04 응급실 패혈증 고위험 팝업

### 1) 목적

- 응급실에서 패혈증 고위험 환자를 즉시 인지하게 한다.

### 2) 표시 조건

- qSOFA 2점 이상

### 3) 팝업 구성

| 요소 | 내용 |
|---|---|
| 팝업 색상 | 빨간색 또는 진한 주황색 강조 |
| 제목 | `패혈증 고위험 환자입니다.` |
| 본문 1 | `수축기혈압 88, 호흡수 26, 의식변화 있음` |
| 본문 2 | `즉시 평가 및 조치가 필요합니다.` |
| 버튼 1 | `상세보기` |
| 버튼 2 | `오더 확인` |
| 버튼 3 | `확인` |

### 4) 동작 규칙

- 활력징후 입력 후 5초 이내 표시
- 동일 단계 경고는 최근 60분 이내 중복 억제
- 상태 악화 시 즉시 재노출 가능

---

## UI-05 패혈증 상세 팝업

### 1) 목적

- 경고 발생 근거와 권고 조치를 함께 보여준다.

### 2) 표시 정보

| 항목 | 예시 |
|---|---|
| qSOFA 점수 | 3 |
| 수축기혈압 | 88 |
| 호흡수 | 26 |
| 의식상태 | Altered |
| 권고 조치 | 혈액배양, 젖산, 항생제 검토 |
| 최근 오더 현황 | 혈액배양 미입력 |

### 3) 버튼

- `혈액배양 오더`
- `젖산 확인`
- `항생제 검토`
- `나중에 보기`

---

## UI-06 운영자 로그 조회 화면

### 1) 목적

- 운영자가 알림 현황과 성과를 모니터링한다.

### 2) 조회 조건

- 기간
- 알림유형
- 진료과
- 사용자
- 처리결과

### 3) 표시 항목

| 항목 | 설명 |
|---|---|
| 알림 발생 건수 | 기간별 총 발생 수 |
| 수용률 | 알림 후 실제 조치로 이어진 비율 |
| 무시율 | 무시 처리 비율 |
| 후속조치율 | 패혈증 알림 후 오더 수행 비율 |
| 오경보 검토 대상 | 임상적으로 재검토 필요한 건수 |

## 5. 예외 처리 설계

| 상황 | UI 처리 방식 |
|---|---|
| 외부검사 결과 등록 존재 | 알림 미표시 또는 참고 배너만 표시 |
| 동일 알림 반복 발생 | 중복 억제 정책에 따라 재노출 제한 |
| CDSS 연계 실패 | `알림 정보를 불러오지 못했습니다` 메시지 표시 |
| 권한 없는 사용자 접근 | 상세 팝업 또는 운영자 화면 접근 차단 |

## 6. 한 줄 정리

이 사용자 인터페이스 설계서의 핵심은  
`당뇨 알림은 진료 흐름을 방해하지 않게, 패혈증 알림은 즉시 대응 가능하게, 그리고 모든 알림은 근거와 후속 액션이 보이도록 설계하는 것`이다.
