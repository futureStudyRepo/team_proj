# DES10 단위시험 케이스 예시

문서명: 단위시험 케이스  
대상 기능:

- 당뇨 환자 추적검사 누락 알림
- 패혈증 위험 알림

## 1. 시험 케이스

| TC ID | 기능 | 입력 조건 | 기대 결과 |
|---|---|---|---|
| UT-01 | 당뇨 관리대상 식별 | 최근 1년 내 E11.9 진단 존재 | 관리대상으로 식별 |
| UT-02 | 당뇨 관리대상 식별 | 진단 없음, 당뇨약 처방 없음 | 관리대상 미식별 |
| UT-03 | HbA1c 누락 알림 | 마지막 HbA1c 검사일 100일 전 | 알림 발생 |
| UT-04 | HbA1c 누락 알림 | 마지막 HbA1c 검사일 30일 전 | 알림 미발생 |
| UT-05 | 외부검사 예외 처리 | 외부검사 등록 있음 | 알림 미발생 |
| UT-06 | qSOFA 계산 | SBP 88, RR 26, 의식변화 있음 | 점수 3, 고위험 알림 |
| UT-07 | qSOFA 계산 | SBP 110, RR 18, 의식정상 | 점수 0, 알림 없음 |
| UT-08 | 중복알림 억제 | 동일 패혈증 경고 30분 전 발생 | 재알림 억제 |
| UT-09 | 상태악화 승급 | 기존 주의 상태에서 젖산 4.2 수신 | 고위험 승급 |
| UT-10 | 로그 저장 | 알림 확인 후 검사 오더 입력 | 처리결과 `ORDERED_TEST` 저장 |

## 2. 한 줄 정리

단위시험의 핵심은  
`규칙과 예외가 코드 수준에서 정확히 동작하는지 확인하는 것`이다.
