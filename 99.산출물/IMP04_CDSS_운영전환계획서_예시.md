# IMP04 운영전환 계획서 예시

문서명: 운영전환 계획서  
대상 시스템: HIS 연계 CDSS

## 1. 목적

- CDSS 기능을 운영환경에 안전하게 배포하고 안정화한다.
- 사용자 교육, 초기 모니터링, 장애 대응, KPI 점검 체계를 준비한다.

## 2. 운영전환 범위

| 구분 | 내용 |
|---|---|
| 1차 오픈 | 당뇨 관리대상 식별, HbA1c 누락 알림, 패혈증 고위험 알림 |
| 포함 시스템 | HIS, EMR, OCS, LIS, NURSING, CDSS Engine |
| 제외 범위 | 안저검사 예약 연계 고도화, 후속조치 재알림 |

## 3. 단계별 전환 계획

| 단계 | 내용 |
|---|---|
| 사전 준비 | 코드 배포, 규칙 초기값 반영, 기준 데이터 적재 |
| 사용자 교육 | 의사/간호사/코디네이터/운영자 교육 |
| 시범 운영 | 외래/응급실 제한 오픈 후 모니터링 |
| 확대 적용 | 안정화 후 대상 부서 확대 |

## 4. 초기 안정화 모니터링

| 항목 | 점검 내용 |
|---|---|
| 알림 발생 수 | 과다 발생 여부 |
| 무시율 | alert fatigue 징후 확인 |
| 조치율 | 알림 후 실제 오더/행동 연결 비율 |
| 성능 | 5초 이내 응답 여부 |
| 장애 | 인터페이스 실패 및 지연 여부 |

## 5. 한 줄 정리

운영전환 계획의 핵심은  
`기능 오픈 자체보다 오픈 후 현장 수용성과 안정화를 체계적으로 관리하는 것`이다.
