# DES09 총괄시험 계획서 예시

문서명: 총괄시험 계획서  
대상 시스템: HIS 연계 CDSS  
대상 기능:

- 당뇨 환자 추적검사 누락 알림
- 패혈증 위험 알림

## 1. 시험 목적

- CDSS 기능이 요구사항과 설계서에 맞게 구현되었는지 검증한다.
- 데이터 연계, 화면 표시, 알림 판정, 후속조치 기록이 정상 동작하는지 확인한다.
- 임상 시나리오 기반으로 환자안전 관점의 위험을 사전 검증한다.

## 2. 시험 범위

| 구분 | 범위 |
|---|---|
| 기능 시험 | 당뇨 관리대상 식별, HbA1c 누락 알림, 패혈증 고위험 알림 |
| 인터페이스 시험 | HIS, EMR, OCS, LIS, NURSING 연계 |
| 화면 시험 | 외래 배너, 상세 팝업, 응급실 고위험 팝업, 운영화면 |
| 성능 시험 | 활력징후 입력 후 5초 이내 알림 표시 |
| 보안 시험 | 권한 없는 사용자 접근 차단 |
| 임상 시나리오 시험 | 실제 업무 흐름 중심 검증 |

## 3. 시험 유형

| 시험 유형 | 목적 | 산출물 연계 |
|---|---|---|
| 단위시험 | 로직별 정확성 검증 | DES10, IMP03 |
| 통합시험 | 시스템 간 연계 검증 | DES11, TES01 |
| 시스템시험 | 사용자 관점 전체 흐름 검증 | DES12, TES02 |
| 인수시험 | 최종 사용자 승인 | TRA06, TRA07 |

## 4. 역할 및 책임

| 역할 | 책임 |
|---|---|
| QA | 시험계획 수립, 수행 관리, 결함관리 |
| 의료정보팀 | 기술 지원, 데이터 연계 점검 |
| 내분비내과 | 당뇨 임상 검증 |
| 응급의학과 | 패혈증 임상 검증 |
| 감염관리실 | 후속조치 정책 및 KPI 검증 |
| 간호부 | 활력징후 입력 및 workflow 검증 |

## 5. 합격 기준

| 항목 | 기준 |
|---|---|
| 중대 결함 | 0건 |
| 주요 기능 성공률 | 100% |
| 실시간 알림 성능 | 95 percentile 기준 5초 이내 |
| 임상 검증 | 주요 시나리오 승인 |
| 보안 | 권한 통제 오류 없음 |

## 6. 시험 일정 예시

| 주차 | 활동 |
|---|---|
| 1주차 | 단위시험 및 결함 수정 |
| 2주차 | 통합시험 수행 |
| 3주차 | 시스템시험 및 성능시험 |
| 4주차 | 인수시험 및 최종 승인 |

## 7. 한 줄 정리

이 총괄시험 계획서의 핵심은  
`기능 정확성뿐 아니라 의료 업무 적합성과 환자안전까지 포함한 검증 구조를 단계적으로 운영하는 것`이다.
