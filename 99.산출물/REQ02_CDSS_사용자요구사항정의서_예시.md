# REQ02 사용자 요구사항 정의서 예시

문서명: 사용자 요구사항 정의서  
대상 시스템: 병원행정시스템(HIS) 연계 CDSS  
대상 기능:

- 당뇨 환자 추적검사 누락 알림
- 패혈증 위험 알림

## 1. 문서 목적

본 문서는 병원행정시스템과 연계되는 CDSS 기능에 대해 사용자 관점의 요구사항을 정의하기 위한 것이다.  
주요 사용자는 의사, 간호사, 코디네이터, 감염관리실, 운영자이며, 본 문서는 기능 요구사항뿐 아니라 화면, 데이터, 예외, 운영 요구까지 포함한다.

## 2. 업무 배경

### 2.1 당뇨 환자 추적검사 누락 알림

- 당뇨 환자는 정기적으로 HbA1c, 신장 합병증 검사, 안저검사 등의 추적검사가 필요하다.
- 현행 업무에서는 의료진이 직접 과거 검사 이력을 확인해야 하므로 누락 가능성이 존재한다.
- 진료 시점에 필요한 검사 누락 여부를 자동으로 알려주면 지속관리 품질을 높일 수 있다.

### 2.2 패혈증 위험 알림

- 응급실 및 병동에서는 감염 의심 환자의 상태 악화를 조기에 인지하는 것이 중요하다.
- 활력징후, 의식상태, 검사결과를 기반으로 한 조기 탐지가 환자안전에 직접 영향을 준다.
- 위험도 기준 충족 시 즉시 알림을 제공하면 초기 대응 시간을 줄일 수 있다.

## 3. 이해관계자

| 구분 | 역할 | 주요 관심사항 |
|---|---|---|
| 내분비내과 의사 | 당뇨 지속관리 | HbA1c 및 합병증 추적검사 누락 여부 |
| 외래 간호사/코디네이터 | 예약 및 안내 지원 | 안저검사, 추가 검사 예약 연계 |
| 응급실 의사 | 패혈증 조기인지 및 처치 | 고위험 환자 즉시 경고 |
| 병동 간호사 | 활력징후 입력 및 재평가 | 재측정 및 보고 흐름 |
| 감염관리실 | 성과관리 및 정책관리 | 경고 후 조치율, 오경보율 |
| 운영자 | 규칙 및 로그 운영 | 임계값 조정, 로그 확인, 장애 대응 |

## 4. 현행 문제점

| 번호 | 현행 문제 | 영향 |
|---|---|---|
| 1 | 당뇨 추적검사 여부를 의료진이 수동 확인 | 검사 누락 발생 가능 |
| 2 | 외부검사 결과 반영 여부가 일관되지 않음 | 불필요한 재처방 및 혼선 |
| 3 | 패혈증 위험 환자를 담당자가 경험에 의존해 식별 | 초기 대응 지연 위험 |
| 4 | 활력징후와 검사결과를 종합적으로 보기 어려움 | 상태 악화 인지 지연 |
| 5 | 알림 발생 후 조치 결과 추적 체계 부족 | 운영성과 분석 어려움 |

## 5. 사용자 요구사항

### 5.1 기능 요구사항

| 요구사항 ID | 사용자 | 요구사항 | 상세 내용 | 우선순위 |
|---|---|---|---|---|
| UR-01 | 내분비내과 의사 | 당뇨 관리대상 환자 식별 | 최근 1년 내 당뇨 진단 또는 당뇨약 처방 이력이 있으면 관리대상으로 자동 식별되어야 한다. | 상 |
| UR-02 | 주치의 | HbA1c 누락 알림 | 최근 90일 이내 HbA1c 결과가 없으면 진료화면에서 알림을 봐야 한다. | 상 |
| UR-03 | 코디네이터 | 안저검사 누락 확인 | 최근 1년 안저검사 기록이 없는 환자를 확인하고 예약을 유도할 수 있어야 한다. | 중 |
| UR-04 | 응급실 의사 | 패혈증 고위험 알림 | qSOFA 기준을 충족하면 즉시 고위험 경고를 받아야 한다. | 상 |
| UR-05 | 감염관리실 | 후속조치 추적 | 패혈증 경고 후 혈액배양, 젖산, 항생제 오더 여부를 추적할 수 있어야 한다. | 중 |
| UR-06 | 운영자 | 알림 로그 조회 | 알림 발생, 확인, 무시, 조치 결과를 로그로 조회할 수 있어야 한다. | 상 |

### 5.2 화면 요구사항

| 요구사항 ID | 화면 | 요구사항 | 상세 내용 |
|---|---|---|---|
| UI-01 | 외래 진료화면 | 당뇨 알림은 배너 형태 | 팝업보다 배너 우선으로 표시하고 상세 버튼을 제공한다. |
| UI-02 | 응급실 화면 | 패혈증 고위험은 팝업 표시 | 고위험 환자는 즉시 인지할 수 있도록 팝업 또는 강조 경고를 표시한다. |
| UI-03 | 상세 팝업 | 발생 근거 표시 | 왜 알림이 떴는지 마지막 검사일, 활력징후 값, 기준조건을 표시한다. |
| UI-04 | 알림 처리 | 무시 사유 입력 | 알림을 무시할 경우 사유를 선택 또는 입력할 수 있어야 한다. |

### 5.3 데이터 요구사항

| 요구사항 ID | 데이터 항목 | 상세 내용 |
|---|---|---|
| DATA-01 | 진단코드 | 당뇨 관리대상 식별에 사용 |
| DATA-02 | 약물 처방이력 | 당뇨약 처방 여부 확인 |
| DATA-03 | HbA1c 결과일/결과값 | 추적검사 누락 판단 기준 |
| DATA-04 | 안저검사 시행일 | 합병증 추적 여부 확인 |
| DATA-05 | 수축기혈압, 호흡수, 의식상태 | qSOFA 판정 기준 |
| DATA-06 | 젖산, 백혈구수, 혈액배양 오더 | 패혈증 중증도 및 후속조치 추적 |

### 5.4 비기능 요구사항

| 요구사항 ID | 항목 | 요구사항 |
|---|---|---|
| NFR-01 | 성능 | 활력징후 입력 후 패혈증 고위험 알림은 5초 이내 표시되어야 한다. |
| NFR-02 | 보안 | 환자정보 및 알림 로그는 권한 있는 사용자만 열람 가능해야 한다. |
| NFR-03 | 감사추적 | 누가 언제 어떤 알림을 받았고 어떻게 처리했는지 저장되어야 한다. |
| NFR-04 | 가용성 | CDSS 장애가 발생해도 HIS 기본 진료 기능은 유지되어야 한다. |

## 6. 예외 및 운영 정책 요구

| 구분 | 내용 |
|---|---|
| 당뇨 예외 | 외부검사 결과가 등록된 경우 누락 알림 대상에서 제외 가능해야 한다. |
| 당뇨 예외 | 완화의료 또는 임종기 환자는 정책에 따라 알림 제외 가능해야 한다. |
| 패혈증 예외 | 중환자실, 소아, 특정 병동은 별도 임계값 정책을 적용할 수 있어야 한다. |
| 공통 운영 | 동일 알림의 과도한 반복을 방지하기 위한 중복 억제 정책이 필요하다. |

## 9. 한 줄 정리

이 사용자 요구사항 정의서의 핵심은  
`당뇨 환자 추적검사 누락과 패혈증 위험을 진료 흐름 안에서 적절한 시점에 알리고, 그 근거와 후속조치를 운영 가능한 수준으로 연결하는 것`이다.
