# TRA02 운영자 지침서 예시

문서명: 운영자 지침서  
대상 사용자:

- 의료정보팀
- 운영팀
- 감염관리실

## 1. 운영자 주요 역할

| 역할 | 설명 |
|---|---|
| 로그 모니터링 | 알림 발생, 무시, 조치 결과 점검 |
| 규칙 점검 | 과다 경고, 미탐 가능성, 문구 적정성 검토 |
| 임계값 변경 관리 | 임상 승인 후 규칙 값 조정 |
| 장애 대응 | 인터페이스 실패, 지연, 로그 누락 처리 |
| KPI 보고 | 월간 운영 성과 보고 |

## 2. 일일 점검 항목

| 항목 | 점검 내용 |
|---|---|
| 인터페이스 수신 상태 | LIS, 활력징후 이벤트 정상 수신 여부 |
| 알림 발생 건수 | 전일 대비 급증 여부 |
| 무시율 | 특정 부서/사용자 편중 여부 |
| 후속조치율 | 패혈증 고위험 후 오더 수행 비율 |
| 장애 로그 | 실패/재처리 여부 |

## 3. 규칙 변경 절차

1. 운영자가 문제 징후를 발견한다.
2. 임상부서와 원인을 검토한다.
3. 변경 필요 시 규칙 조정안을 작성한다.
4. 임상 승인 후 운영화면에 반영한다.
5. 적용 후 1주일간 모니터링한다.

## 4. KPI 예시

| KPI | 설명 |
|---|---|
| HbA1c 시행률 | 알림 대상 중 실제 검사 시행 비율 |
| 패혈증 경고 후 조치율 | 고위험 알림 후 권고 조치 비율 |
| 무시율 | 알림이 무시된 비율 |
| 오경보 검토 건수 | 재검토가 필요한 알림 수 |

## 5. 한 줄 정리

운영자 지침서의 핵심은  
`알림 시스템을 단순 유지하는 것이 아니라, 운영 데이터에 기반해 지속적으로 조정하고 개선하는 것`이다.
