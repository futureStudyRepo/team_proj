# ANA06 인터페이스 정의서 예시

문서명: 인터페이스 정의서  
대상 시스템: 병원행정시스템(HIS) 연계 CDSS  
대상 기능:

- 당뇨 환자 추적검사 누락 알림
- 패혈증 위험 알림

## 1. 문서 목적

본 문서는 CDSS 기능 구현을 위해 병원 내/외부 시스템과 연계되어야 하는 인터페이스를 정의하기 위한 것이다.  
CDSS는 데이터 연계 품질에 따라 정확도와 운영 안정성이 크게 좌우되므로, 각 인터페이스의 입력 데이터, 연계 시점, 전송 방식, 실패 대응을 명확히 정의해야 한다.

## 2. 연계 대상 시스템

| 시스템 | 설명 | CDSS에서의 활용 |
|---|---|---|
| HIS | 환자 기본정보, 방문정보, 사용자정보 제공 | 환자 식별, 화면 연계, 로그 사용자 식별 |
| EMR | 진단, 문진, 진료기록 제공 | 당뇨 진단, 감염 의심 정보, 외부검사 등록 |
| OCS | 처방 및 오더 정보 제공 | 당뇨약 이력, 혈액배양/젖산/항생제 오더 확인 |
| LIS | 검사 결과 제공 | HbA1c, 미세알부민뇨, 젖산, 백혈구수 결과 수신 |
| NURSING | 활력징후 및 간호평가 제공 | 수축기혈압, 호흡수, 체온, 심박수, 의식상태 조회 |
| CDSS Engine | 규칙 판정 및 알림 처리 | 연계 데이터 기반 규칙 실행, 알림 생성 |
| ADMIN/운영화면 | 규칙 관리 및 로그 조회 | 임계값 조정, 운영 지표 조회 |

## 3. 인터페이스 목록

| 인터페이스 ID | 송신 시스템 | 수신 시스템 | 인터페이스명 | 목적 | 방식 | 실시간 여부 |
|---|---|---|---|---|---|---|
| IF-01 | HIS | CDSS | 환자기본정보 조회 | 환자ID, 방문ID, 진료과, 담당의 조회 | API/조회 | 실시간 |
| IF-02 | EMR | CDSS | 진단정보 조회 | 당뇨 진단, 감염 의심 진단 조회 | DB/API | 실시간 또는 준실시간 |
| IF-03 | OCS | CDSS | 처방 및 오더 정보 조회 | 당뇨약 처방, 혈액배양/항생제 오더 조회 | DB/API | 실시간 또는 준실시간 |
| IF-04 | LIS | CDSS | 검사결과 수신 | HbA1c, 젖산, 백혈구수 등 결과 수신 | HL7/API/배치 | 준실시간 |
| IF-05 | NURSING | CDSS | 활력징후 이벤트 수신 | 활력징후 저장 시 qSOFA 재판정 | 이벤트/API | 실시간 |
| IF-06 | CDSS | HIS UI | 알림 결과 제공 | 배너/팝업 표시 데이터 전달 | 내부 API | 실시간 |
| IF-07 | CDSS | 로그/운영화면 | 알림 로그 적재 | 알림 발생/확인/무시/조치 저장 | DB/내부 API | 실시간 |
| IF-08 | ADMIN | CDSS | 규칙 설정 반영 | 임계값, 문구, 활성화 상태 반영 | 내부 API | 준실시간 |

## 4. 인터페이스 상세 정의

---

## IF-01 환자기본정보 조회

### 1) 목적

- 환자 차트 열람 시 CDSS가 대상 환자를 정확히 식별하기 위해 사용한다.

### 2) 주요 데이터 항목

| 항목 | 설명 | 예시 |
|---|---|---|
| 환자ID | 병원 내부 환자 식별자 | P20260416001 |
| 방문ID | 외래/입원/응급 방문 식별자 | OP202604160101 |
| 진료과 | 현재 방문 진료과 | 내분비내과 |
| 담당의ID | 현재 진료 담당자 | DR0142 |
| 방문구분 | 외래/입원/응급 | 외래 |

### 3) 호출 시점

- 사용자가 환자 차트를 열 때
- 알림 화면을 다시 조회할 때

### 4) 실패 대응

- 환자기본정보 조회 실패 시 CDSS 판정을 수행하지 않는다.
- 사용자에게 `알림 정보를 불러오지 못했습니다` 메시지를 표시한다.

---

## IF-02 진단정보 조회

### 1) 목적

- 당뇨 관리대상 환자 식별 및 감염 의심 여부 판단에 사용한다.

### 2) 주요 데이터 항목

| 항목 | 설명 | 예시 |
|---|---|---|
| 진단코드 | ICD 또는 병원 내부 진단코드 | E11.9 |
| 진단명 | 진단 텍스트 | 제2형 당뇨병 |
| 진단일 | 최초 또는 최근 진단일 | 2025-11-10 |
| 감염 의심 플래그 | 감염 관련 진단/문진 여부 | Y |

### 3) 활용 규칙

- 최근 1년 내 E10-E14 진단이 있으면 당뇨 관리대상 후보로 본다.
- 감염 의심 관련 진단 또는 기록이 있으면 패혈증 감시대상 판단 보조자료로 사용한다.

---

## IF-03 처방 및 오더 정보 조회

### 1) 목적

- 당뇨약 처방 이력 확인, 패혈증 후속조치 여부 확인에 사용한다.

### 2) 주요 데이터 항목

| 항목 | 설명 | 예시 |
|---|---|---|
| 약품코드 | 처방 약품 코드 | DMMED0231 |
| 약품명 | 처방 약품명 | 메트포르민 |
| 처방일시 | 처방 입력 시각 | 2026-04-16 09:14 |
| 오더코드 | 검사/치료 오더 코드 | BCUL001 |
| 오더상태 | 처방 입력/취소 여부 | Ordered |

### 3) 활용 규칙

- 최근 1년 내 당뇨약 처방 이력이 있으면 당뇨 관리대상 후보로 본다.
- 패혈증 알림 후 혈액배양, 젖산, 항생제 오더 여부를 확인한다.

---

## IF-04 검사결과 수신

### 1) 목적

- 당뇨 추적검사 누락 여부 및 패혈증 중증도 판단에 필요한 검사결과를 수신한다.

### 2) 주요 데이터 항목

| 항목 | 설명 | 예시 |
|---|---|---|
| 검사코드 | 검사 종류 식별 | HBA1C001 |
| 검사명 | 검사명 | HbA1c |
| 결과값 | 수치 결과 | 8.4 |
| 결과단위 | 수치 단위 | % |
| 결과일시 | 최종 결과 시각 | 2026-03-20 08:15 |

### 3) 활용 규칙

- HbA1c 결과일이 최근 90일 이내면 누락 알림을 표시하지 않는다.
- 젖산 결과가 특정 기준 이상이면 패혈증 위험도 참고자료로 활용한다.

### 4) 실패 대응

- 결과 수신 지연 시 마지막 정상 수신 시각을 운영화면에 표시한다.
- 검사결과 누락 여부는 운영 로그에서 재확인 가능해야 한다.

---

## IF-05 활력징후 이벤트 수신

### 1) 목적

- 활력징후 입력 직후 패혈증 qSOFA 규칙을 재평가하기 위해 사용한다.

### 2) 주요 데이터 항목

| 항목 | 설명 | 예시 |
|---|---|---|
| 수축기혈압 | qSOFA 기준 항목 | 88 |
| 호흡수 | qSOFA 기준 항목 | 26 |
| 체온 | 보조 판단 항목 | 38.7 |
| 심박수 | 보조 판단 항목 | 122 |
| 의식상태 | qSOFA 기준 항목 | Altered |
| 입력시각 | 저장 시점 | 2026-04-16 10:05 |

### 3) 처리 시점

- 활력징후 저장 이벤트 발생 즉시
- 목표 반응속도: 5초 이내 알림 표시

### 4) 실패 대응

- 이벤트 수신 실패 시 배치 재판정 또는 수동 새로고침이 가능해야 한다.
- 장애 발생 내역은 운영 로그에 기록한다.

---

## IF-06 알림 결과 제공

### 1) 목적

- CDSS 판정 결과를 HIS 사용자 화면에 배너/팝업 형태로 표시한다.

### 2) 주요 데이터 항목

| 항목 | 설명 | 예시 |
|---|---|---|
| 알림유형 | 당뇨/HbA1c/패혈증고위험 등 | SEPSIS_HIGH |
| 알림문구 | 사용자에게 보여줄 메시지 | 패혈증 고위험입니다. 즉시 평가가 필요합니다. |
| 발생근거 | 수치 및 기준 설명 | SBP 88, RR 26, 의식변화 |
| 우선순위 | 강조 수준 | High |
| 상세링크 | 추가 확인 버튼 | 상세보기 |

### 3) 화면 정책

- 당뇨 추적검사 누락은 배너 우선
- 패혈증 고위험은 팝업 또는 강조 알림 우선
- 상세보기에서 발생 근거를 반드시 제공

---

## IF-07 알림 로그 적재

### 1) 목적

- 감사추적과 운영 KPI 계산을 위해 알림 관련 이벤트를 저장한다.

### 2) 주요 데이터 항목

| 항목 | 설명 | 예시 |
|---|---|---|
| 로그ID | 로그 식별자 | LOG202604160001 |
| 알림유형 | 알림 분류 | HBA1C_MISSING |
| 발생시각 | 알림이 생성된 시각 | 2026-04-16 09:16 |
| 사용자ID | 알림을 본 사용자 | DR0142 |
| 처리결과 | 확인/무시/오더입력 | ORDERED_TEST |
| 무시사유 | 무시한 경우 사유 | 외부검사 예정 |

### 3) 활용

- 알림 수용률, 무시율, 후속조치율 분석
- 감사 및 장애 분석 자료

---

## IF-08 규칙 설정 반영

### 1) 목적

- 운영자가 관리화면에서 임계값과 문구를 조정할 수 있게 한다.

### 2) 주요 데이터 항목

| 항목 | 설명 | 예시 |
|---|---|---|
| 규칙ID | 규칙 식별자 | RULE-SEPSIS-QSOFA |
| 임계값 | 기준값 | 2 |
| 활성여부 | 사용 여부 | Y |
| 문구 | 표시 메시지 | 패혈증 고위험입니다. 즉시 평가가 필요합니다. |
| 적용일시 | 반영 시점 | 2026-04-16 00:00 |

### 3) 운영 정책

- 임상 승인 후 반영
- 변경 이력 저장
- 이전 버전 조회 가능

## 5. 인터페이스 공통 요구사항

| 구분 | 요구사항 |
|---|---|
| 보안 | 환자정보와 로그는 권한 있는 사용자만 접근 가능해야 한다. |
| 성능 | 실시간 알림 인터페이스는 5초 이내 응답해야 한다. |
| 안정성 | CDSS 장애가 발생해도 HIS 기본 진료 기능은 유지되어야 한다. |
| 감사 | 인터페이스 실패 및 재처리 이력을 기록해야 한다. |
| 정합성 | 코드 매핑, 단위, 날짜 기준이 일관되어야 한다. |

## 6. 한 줄 정리

이 인터페이스 정의서의 핵심은  
`CDSS가 어떤 시스템으로부터 어떤 데이터를 언제 받아서 어떤 알림과 로그로 연결하는지 명확히 정의하여, 의료 현장에서 신뢰할 수 있는 연계 구조를 만드는 것`이다.
